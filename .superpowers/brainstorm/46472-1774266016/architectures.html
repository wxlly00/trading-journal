3 Architectures possibles

Choisir l'approche qui correspond le mieux à ton workflow

A

Monolithe cloud

L'EA envoie les trades directement vers l'API déployée sur internet (Railway/Render). Tout est dans le cloud.

MT5 EA → Internet → FastAPI (Railway)
                          ↓
React (Vercel) ←──── PostgreSQL (Neon)

Avantages

  • Setup unique, tout cloud
  • Accessible partout sans config locale

Inconvénients

  • EA doit avoir internet pour logger
  • Latence réseau sur chaque trade
B

Collecteur local + Cloud ⭐ Recommandée

Un petit service tourne sur ton Mac pour recevoir l'EA. Il sync vers Supabase. Le dashboard est accessible partout.

MT5 EA → localhost:8000 → Supabase (PostgreSQL)
                                   ↓
                      React (Vercel) ←────┘

Avantages

  • EA push local = fiable, zéro latence
  • Dashboard accessible depuis téléphone
  • Supabase = free tier généreux

Inconvénients

  • Service local doit tourner quand MT5 est ouvert
  • Deux composants à gérer
C

Tunnel Cloudflare

Même logique que B, mais le service local est exposé via un tunnel Cloudflare (cloudflared) — pas besoin de cloud pour le backend.

MT5 EA → localhost:8000 → SQLite (local)
                ↓
cloudflared → ton-app.trycloudflare.com
                ↑
React (Vercel)

Avantages

  • Zéro cloud payant
  • Données restent sur ton Mac

Inconvénients

  • Mac doit être allumé pour accès mobile
  • URL change à chaque restart (version gratuite)
